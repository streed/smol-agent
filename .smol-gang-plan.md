# smol-gang Plan

## Prompt

Let's build a web ui for the agent that a user of the agent can use to control this. This should include a api layer, a ui layer written in react, and finally the ui should be real time as the agent does work.

## Tasks

- **agent-event-system** (wave 0): Create an event emitter/observer pattern in the agent core to broadcast state changes, progress updates, thinking status, and output. Define event types (AGENT_START, AGENT_STOP, THINKING, TOOL_USE, OUTPUT, ERROR) in /agent/events.py. Modify existing agent loop to emit events at key points.
- **api-foundation** (wave 0): Set up FastAPI backend server with basic routing structure, CORS middleware for frontend communication, health check endpoint at GET /api/health, and project structure under /api directory. Include pyproject.toml dependencies for fastapi, uvicorn, and python-socketio.
- **react-app-init** (wave 0): Initialize React frontend with TypeScript using Vite. Set up project structure under /frontend directory with src/components, src/hooks, src/api, and src/types folders. Configure proxy to backend API in vite.config.ts. Create base App.tsx layout with routing setup using react-router-dom.
- **agent-control-api** (wave 1): Implement REST API endpoints for agent control: POST /api/tasks to submit new task, POST /api/tasks/:id/start to begin execution, POST /api/tasks/:id/stop to halt, POST /api/tasks/:id/pause to pause. Create task manager to track task state and queue. Include request/response schemas with Pydantic models.
- **dashboard-components** (wave 1): Build main dashboard UI components: AgentStatusCard showing current agent state (idle/running/thinking), ProgressBar for task completion, ConnectionStatus indicator for WebSocket connection state. Create layout with sidebar navigation and main content area. Style with CSS modules or Tailwind.
- **websocket-server** (wave 1): Implement WebSocket endpoint at /ws on the FastAPI server using python-socketio or websockets library. Handle client connections, disconnections, and message routing. Create connection manager to track active clients. Expose method to broadcast messages to all connected clients.
- **realtime-broadcast** (wave 2): Connect the agent event system to WebSocket server. When agent emits events (THINKING, TOOL_USE, OUTPUT, ERROR), broadcast them to all connected WebSocket clients. Ensure event payload includes all relevant context (message, tool name, progress percentage). Handle graceful degradation if no clients connected.
- **task-input-ui** (wave 2): Create task submission form component with textarea for task description, model selection dropdown, and submit button. Implement form validation. Connect to POST /api/tasks endpoint. Show loading state during submission. Display created task confirmation with task ID. Handle and display API errors.
- **live-output-display** (wave 3): Build scrolling output display component that connects to WebSocket and renders real-time agent output. Display different event types with distinct styling (thinking italic, errors red, tool calls highlighted). Auto-scroll to show latest output. Include pause/resume scroll toggle. Handle reconnection on WebSocket disconnect.
